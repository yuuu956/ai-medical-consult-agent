# AI 医疗问诊 Agent（非诊断）

## 一、项目简介

这是一个基于 AI Agent 思路设计的医疗问诊辅助 Demo。

项目目标不是替代医生，也不提供医学诊断，而是通过多轮问答完成以下任务：

1. 收集用户症状信息
2. 判断是否存在明显高风险信号
3. 推荐可能适合就诊的科室
4. 给出非诊断性质的就医建议
5. 展示 AI 医疗场景中的 Risk Control 设计

## 二、产品定位

本项目定位为：

> 面向普通用户的「AI 就医前问诊助手」

它可以帮助用户在就医前整理症状、识别紧急风险、了解可能就诊方向，但不能替代医生诊断。

## 三、核心流程

用户输入症状

↓

↓

结构化整理症状

↓

↓

↓

就医建议

## 四、核心功能模块

### 1. 症状收集

收集用户的主要症状、持续时间、严重程度、伴随症状、既往病史等信息。

### 2. 风险判断

识别是否存在需要立即就医或急诊处理的高风险信号，例如：

- 胸痛
- 呼吸困难
- 意识模糊
- 持续高热
- 剧烈头痛
- 严重外伤
- 大量出血

### 3. 科室推荐

根据用户症状，推荐可能适合咨询的科室，例如：

- 呼吸内科
- 消化内科
- 心内科
- 神经内科
- 皮肤科
- 耳鼻喉科
- 急诊科

### 4. 就医建议

给出非诊断性质的建议，例如：

- 建议尽快线下就医
- 建议优先挂某科室
- 建议记录症状变化
- 建议携带既往检查结果

## 五、Risk Control 设计

本项目特别强调医疗 AI 的风险控制：

1. 明确声明不提供诊断
2. 不输出具体疾病结论
3. 遇到高风险症状时优先建议急诊或立即就医
4. 不推荐具体药物剂量
5. 不替代医生判断
6. 对不确定情况进行保守处理
7. 输出内容包含安全提醒
